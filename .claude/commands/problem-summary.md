请执行以下流程并输出：

1. 检查 `docs/` 是否存在；若不存在则创建。
2. 检查 `/docs/problem.md` 是否存在；若不存在则新建并写入文件头。
3. 在写入前先快速检索 `problem.md` 是否已有相似问题：
   - 至少用 2 组关键词（错误码/异常类型/模块名）
   - 推荐关键词模板：
     - 错误码/异常类型 + 模块名
     - 报错关键字 + 文件名/路径
     - 失败动作 + 依赖名/版本
   - 如有相似记录，列出标题与日期并说明差异
4. 生成并追加一次“问题记录”（使用标准模板）。

标准模板必须包含：

- 问题描述（简明具体）
- 影响范围
- 根因
- 解决方案（分步骤）
- 核心修改/关键代码（路径 + 关键片段）
- 验证方式（命令/步骤）
- 参考资料（如有 URL）

追加写入时使用时间戳标题，例如：

```
## 2026-01-18 问题标题
```
