# 强依赖复用要求

## 允许的依赖接入方式

- 包管理器安装（例如 uv / pip / npm / yarn / pnpm）
- 远程可用的源码/制品仓库（Git URL、私有制品仓库、镜像仓库）

> 不依赖本地固定路径，需适配服务器部署与 CI 环境。

## 强制约束

- 必须使用完整、生产级实现
- 禁止复制依赖代码到当前项目再修改
- 禁止对依赖模块做裁剪、重写或降级封装

## 功能完整性

- 被调用能力必须来自依赖库真实实现
- 禁止 mock / stub / demo 替代
- 若依赖已提供功能，禁止自行重写

## 职责边界

- 当前项目仅承担：流程编排、模块组合、参数组织、输入输出适配
- 明确禁止：重写算法、重写数据结构、拆出复杂逻辑自写

## 可验证性

- 所有导入模块必须在运行期真实参与执行
- 避免路径遮蔽与重名模块
- 禁止“只导入不用”的伪集成

## 输出要求

- 明确标注哪些能力来自外部依赖
- 不生成依赖库内部实现代码
- 仅生成最小必要的胶水与业务逻辑
